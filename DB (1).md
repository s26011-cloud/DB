# DB

# <h2>데이터베이스란
정보를 효율적으로 관리, 접근할 수 있도록 체계화한 시스템이다.

# <h2>대표적인 3가지 요소

- 테이블: 데이터베이스에서 가장 기본이 되는 구조 행과 열의 형태로 저장하는 기본 단위이다.

- 필드 & 레코드: 필드란 테이블의 열을 의미하고 레코드란 테이블의 행을 의미한다.

- 스키마: 데이터베이스의 구조를 정의하는 청사진.

# <h2> 데이터베이스는 어떻게 관리되는가?
데이터베이스는 관리 시스템(DBMS)를 통해 관리된다.


- DBMS: 데이터베이스를 생성, 관리, 유지 보수하는 소프트웨어

- RDBMS: 데이터를 구조화된 테이블 형식으로 저장하고 관리하며 데이터베이스 관리 무결성 유지, 보안 등의 기능을 제공.

- NoSQL DBMS: RDBMS의 제약을 극복하기 위해 개발된 프로그램으로
고정된 스키마 없이 다양한 모델을 지원하며, 대규모 데이터를 높은 성능으로 처리하기 위헤 등장.

# <h2> 데이터베이스의 특징

- 실시간 접근성: 수시적이고 비정형적인 질의에 대해 즉시 처리하여 응답함.

- 계속적인 변화: 데이터의 삽입, 삭제 갱신으로 항상 정확한 최신 데이터를 유지한다.

- 동시 공유: 여러 사용자가 동시에 같은 데이터에 접근하여 사용할 수 있다.

- 내용에 의한 참조: 데이터의 위치나 주소가 아닌, 데이터가 가진 값을 기반으로 데이터를 찾는다.

- 데이터의 논리적-물지적 독립성: 응용 프로그램과 데이터베이스가 분리되어, 저장 구조가 변경되어도 프로그램에 영향을 주지 않습니다.

# <h2> 데이터 베이스 흐름 
- 1단계: 데이터 생성 및 입력

- 2단계: 애플리케이션 처리

- 3단계:
DBMS 전달

- 4단계:
데이터베이스 저장

- 5단계:
커밋 및 결과 반환
